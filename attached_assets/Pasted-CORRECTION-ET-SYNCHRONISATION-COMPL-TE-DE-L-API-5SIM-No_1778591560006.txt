CORRECTION ET SYNCHRONISATION COMPLÈTE DE L’API 5SIM

Nous allons corriger et améliorer complètement le système de synchronisation avec l’API 5SIM.

Actuellement, la synchronisation ne fonctionne pas correctement.

Les services, les pays, les prix, les stocks et les disponibilités ne sont pas correctement récupérés ou mis à jour.

Je veux une synchronisation professionnelle, stable et en temps réel avec 5SIM.

---

OBJECTIF PRINCIPAL

Tout ce qui vient de l’API 5SIM doit être synchronisé automatiquement et correctement dans le système.

Cela inclut :

- les pays
- les services
- les prix
- les stocks
- les disponibilités
- les nouveaux services
- les services supprimés
- les changements de statut

Tout doit être mis à jour automatiquement dans la base de données et dans le panneau administrateur.

---

SYNCHRONISATION COMPLÈTE 5SIM

Le système doit :

Synchroniser automatiquement :

- tous les pays disponibles
- tous les services disponibles
- tous les prix
- tous les stocks
- tous les opérateurs
- tous les statuts

Détecter automatiquement :

- les nouveaux services
- les nouveaux pays
- les services supprimés
- les changements de prix
- les changements de stock

---

SYNCHRONISATION EN TEMPS RÉEL

Quand 5SIM :

- ajoute un nouveau service
- modifie un prix
- modifie un stock
- retire un service
- ajoute un nouveau pays

Le système doit automatiquement mettre à jour les données dans le site sans bug.

---

IMPORTANT : PRIX PERSONNALISÉS ADMIN

Même après synchronisation 5SIM, les prix personnalisés définis dans le panneau admin ne doivent jamais être écrasés.

Exemple :

Prix API 5SIM :
WhatsApp USA → 0.8$

Prix défini dans mon panneau admin :
WhatsApp USA → 1200 FCFA

Après synchronisation :

Le système doit conserver mon prix personnalisé admin.

---

GESTION DES PRIX PAR PAYS ET PAR SERVICE

Je dois pouvoir fixer les prix manuellement dans le panneau admin selon :

- le pays
- le service

Exemple :

USA
→ WhatsApp → 1200 FCFA
→ Telegram → 1500 FCFA
→ Snapchat → 900 FCFA

France
→ WhatsApp → 1800 FCFA

Le système doit afficher automatiquement le bon prix côté utilisateur.
